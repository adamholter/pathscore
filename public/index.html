Methodology

How PathScore works

A transparent, reproducible framework for comparing SVG generation quality across language models.

The pipeline

Prompt Dataset
→
N × M Generations
→
Pairwise Judging
→
ELO Rankings

For N models and M prompts, PathScore generates N×M SVGs in parallel. It then creates all N×(N-1)/2 unique pairings per prompt, randomly assigning which generation appears as Model A and which as Model B to eliminate position bias. All pairings are judged concurrently.

Position bias elimination

VLMs can systematically prefer the first or second image shown. PathScore randomizes A/B positions for each pair so no model is consistently advantaged. With multiple judge runs, positions are re-randomized each time and scores aggregated.

ELO rating system

Standard ELO starting at 1000 with K=32. Comparisons are processed chronologically, so rankings shift live during a run.

EA = 1 / (1 + 10^((RB − RA) / 400))

Win: R'A = RA + 32 × (1 − EA)
Loss: R'A = RA + 32 × (0 − EA)
Tie: R'A = RA + 32 × (0.5 − EA)

Comparison count

For N models, M prompts, R judge runs:

Total comparisons = M × N × (N − 1) / 2 × R

Example: 5 models, 10 prompts, 1 judge run = 100 comparisons. Plan API costs accordingly.

Prompt categories

PathScore ships with defaults across multiple categories. AI-assisted generation creates targeted prompts from a natural language description.

Reasoning effort

For models supporting chain-of-thought, configure reasoning effort per-model: low, medium, or high. Compare the same base model at different reasoning budgets within a single run.

Data portability

Every run exports as JSON (all configs, SVGs, comparisons, metadata). Import to continue interrupted runs or share results. The HTML export produces a standalone shareable report. PDF via browser print.
